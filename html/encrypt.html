  CryptoSticky Encryption

   CryptoSticky has a state of the art encryption facility. The stickies are encrypted in-line, where the original content is replaced by the encrypted content. The encrypted content is converted to numbers, so the cipher text stays in text form, which gives the encryption facility  tremendous versatility. One can encrypt text from any document, paste it back, and the content is safe. The plain text approach is so versatile, that one can copy text from any document (like Word or Open Office) or from any email program (like Thunder Bird), encrypt it, and paste it back to its original place. There is even a one button or one keystroke operation to encrypt / decrypt  the clipboard.

   The encryption was designed to be super safe, to allow you to safely store bank account information, site login details, site passwords, or any other sensitive personal information. The encrypted information cannot be seen by intruders, even if your computer is compromised, as the cipher text has an extreme high resistance to crypt analysis. In addition to that, no record of the encryption keys are kept, there is nothing to recover or attack.

Encrypting  a sticky:
  To encrypt a sticky, right click on the body of the sticky. The right click context menu (pictured to the right) shows, with commands for encryption / decryption. Clicking on the 'Encrypt Sticky' command will prompt you for an encryption key to be used.  The encryption key needs to be entered twice, to make sure the entry is the intended value. Upon successful entry, the text of the sticky is encrypted with that key. The key is then discarded, no record of it is kept in any form. Every sticky can have its own unique key, or they all can have the same key, CryptoSticky does not enforce any key rules, and has no key management. This makes it extremely secure, as there is no key to attack.

   To decrypt a sticky, activate the command 'Decrypt Sticky' by clicking on it. The same password dialog appears, prompting you twice for the decryption key. Upon successful key entry, CryptoSticky will decrypt your cipher text, and place it back into its original place. If the wrong key  is entered, the decryption hash will not match, CryptoSticky cannot decrypt the text, and will leave it untouched. There are no limits imposed on the number of tries.  One can encrypt the text of the sticky multiple times, there is no facility to check for double encryption. The original text can be recovered by executing the same decryption in reverse order. One important warning. If you forget the encryption key for a sticky, the text in not recoverable, and it is lost forever. It is a price one pays for higher security.
Encrypting the clipboard:

  To encrypt the clipboard, press the  'Encrypt Clipboard' button on the CryptoSticky's main panel. CryptoSticky will fetch the contents of the clipboard, encrypt it, convert it back into text, and paste it back onto the clipboard in place of the old content. This is a very versatile encryption process, as practically every program in existence has a clipboard text facility. This way, CryptoSticky can extend your favorite program.

   For example, highlighting some text in Word, copying it to the clipboard, encrypting it with CryptoSticky, and pasting it back in to Word, will create a secure portion of that document.


 
 To the left, is a screen shot of Microsoft Word with the original text, the cipher text, then the decrypted original text. We did this by typing the original text into Word, coping it to clipboard, encrypting it with CryptoSticky, and pasted back into Word. Had we pasted it back in place of the original selection, that portion of the document would be encrypted for good.

  To reverse the encryption process, one would need to copy the cipher text to  the clipboard, decrypt it with CryptoSticky.  CryptoSticky will only decrypt the cipher text when someone enters the correct key, the same key as was used in the encryption process.

  
   If all this sounds a little bit too complex, the reality of it can be very simple. All the encryption operations an be executed with a couple of keystrokes and /or  button presses. (Ctrl-C - Encrypt - Ctrl-V)

    Alternatively, all aforementioned operations can be accomplished from the keyboard only. Here is how. The CryptoSticky installation creates a global keystroke "Control-Alt-S" to start the CryptoSticky program. So while you are in any program just select the contents you intend to encrypt. Copy it  to the clipboard (Ctrl-C), and then press the "Control-Alt-S" hot key. CryptoSticky will start (or show if started already) and then you can use the the hot keys Alt-R to Encrypt the clipboard, or Alt-T to Decrypt the clipboard. Alt-S (or Alt-X) will dismiss the CryptoSticky program, and you are back to the original context with the encrypted clipboard, which you can paste back into your document. (Ctrl-V)

   Before you encrypt real text, make sure you go through the cycle of Encryption / Decryption and get familiar with the operation of CryptoSticky. Be forewarned, that if you forget the key or damage the cipher, there is no known recovery method for your original content. (Not even with supercomputers.)

  How secure is CryptoSticky? Very. We deployed state of the art encryption, with more bit combinations than grains of sand in the universe. No key management is implemented, there are no keys stored, and because there are no keys anywhere, there is nothing to attack. (The drawback is you have to remember your keys ... )
  We also created a custom password dialog, that fades the 'CryptoSticky' yellow watermark in and out slowly. This prevents 'spoof' dialogs, as the fading in and out of the alpha blending cannot be duplicated without significant effort.

  If the 'CryptoSticky' yellow watermark is not fading in and out, do not enter any passwords, as you are most likely looking at a compromised system. Be sure to observe this the first time you use it, so you know what to look for.

  There are key logging programs which can record keystrokes, please make sure your system is in good order, and free of any parasitic programs. A good virus checking program should be regularly deployed 


   About the encryption algorithm.  BluePoint Encryption is a simple, secure, state of the art block encryption algorithm. It requires minimal processing power and has a low memory footprint. The algorithm produces an even bit distribution, even when its fed with small data or homogeneous content. This makes it highly resilient against crypt analysis, and an ideal candidate where safe, secure and customized encryption is needed.


   In testing Bluepoint, one of the exceptional properties we found was that in the clear text change, the change propagated to every byte of the cipher text, with no predictable bit distribution. We fed the Bluepoint algorithm various clear text samples and examined the cipher text. It had the most bit-propagation randomness of any algorithm we tested.
 
  The cipher text is editable, it is oblivious to white space  changes. (space-tab-newline) For example, when you paste cipher text into a word document, it can be edited to match the line length or the surrounding text. A warning here is in order - if you change the cipher text and modify even one significant character - your text will be unrecoverable. You have been warned.
